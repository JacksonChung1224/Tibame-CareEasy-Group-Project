# 機構端差異比對規則 v4
比對引擎：`src/utils/reconcile.js`（兩輪配對）。回歸測試：`reconcile.test.mjs`。

**統一語義：裁決 = 選擇哪一邊的資料為真；選中的一邊沒有該筆 → 該筆不成立、不列入核銷明細。**

| 狀態 | 採紙本（含義） | 採系統（含義） | 預設 |
|---|---|---|---|
| D3 數量差異 | 以紙本值申報 | 以排班值申報 | 無預設・必選 |
| D4 項目差異 | 以紙本碼申報 | 以排班碼申報 | 無預設・必選 |
| D1 排班有紙本無 | 紙本為真=未執行 → **不列入**，原因必填 | 排班為真=已執行 → **以排班值列入**，備註選填 | 無預設・必選 |
| no_schedule 紙本有排班無 | 紙本為真=已執行 → **以紙本值列入** | 排班為真=無此服務 → **不列入** | 無預設・必選 |

> 裁決語義 2026-07-15 簽核；無預設化變更 2026-07-20 複核

原則：差異不預設立場，由督導逐筆裁決採紙本或採系統；未裁決不得匯出，確保每一筆應核銷金額都經人工確認後入帳。
同日剩餘雙邊各 ≥2 筆時不做推測配對，各自成 D1／no_schedule 交人工裁決。
